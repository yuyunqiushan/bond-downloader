# 专项债券资料下载器 —— 重构方案

> 制定时间：2026-07-07
> 状态：待实施

## 核心目标

让工具**跑得快、断了能续、进度看得清、日常用着顺手、界面不丑**。

## 使用场景

- 主要来源：chinabond.com.cn 地方债栏目（新版 JSON SPA 接口，docPubUrl 形式），少量 celma.org.cn / 旧版 chinabond 页面
- 每次 1~几条 URL，每条下 ~30 个附件
- 附件：几 MB~几十 MB 的 PDF + 几百 MB~1 GB 的 RAR/ZIP
- 合计约百来个文件/次
- 自用为主，打包 exe 以后再说

## 任务清单

### C 线：下载引擎重构（核心）

| ID | 任务 | 关键内容 |
|---|---|---|
| C-1 | 重写下载核心引擎 | 文件级并发 3 个（可配）；≥50MB 才启用 8 线程分块；小文件单线程直下；HEAD 失败用 Range 兜底；I/O 锁按时间/数据量节流；全局连接数限制；复用 Session；超时 30s/300s |
| C-2 | 跨会话断点续传 | `.download_state.json` 持久化；chunk 存 `.part/` 目录；ETag/Last-Modified 校验；启动时检测未完成任务提示恢复；合并后大小校验 |
| C-3 | 失败重试与错误隔离 | 小文件/chunk 自动重试 3 次（指数退避）；单 URL/文件失败不中断整体；失败清单+一键重试；错误日志写隐藏文件 |
| C-4 | 重构 GUI 进度显示 | 两级进度（总+活动文件区）；总速度+ETA；任务队列状态区分；失败清单分组 |
| C-5 | 暂停/继续 + 运行中追加 URL | 暂停标志；chunk 完成后停止；保存状态；追加 URL 立即调度 |
| C-6 | 修复 parsers.py 现有 bug | 分支顺序 bug；celma 重复请求；新旧页面逻辑合并；改为"提交任务给下载器"模式 |

### B 线：使用体验优化

| ID | 任务 | 关键内容 |
|---|---|---|
| B-1 | 记住用户设置 | 配置存同目录 `.config.json`；记住下载目录、窗口大小位置、并发参数、文件类型过滤 |
| B-2 | URL 列表管理改进 | 下完不清空；粘贴自动去重；导入/导出 .txt；拖拽导入；格式校验；双击复制 |
| B-3 | 下载完成快捷操作 | "打开下载目录"按钮；右键打开文件/所在文件夹；双击完成项打开文件 |
| B-4 | 恢复未完成下载入口 | 启动时检测 `.download_state.json` 提示恢复 |

### D 线：界面与形态

| ID | 任务 | 关键内容 |
|---|---|---|
| D-1 | Tkinter 界面美化 | 高 DPI 修复；微软雅黑字体；标题"专项债券资料下载器"；1000x700 默认+记住位置；.ico 图标；文件类型过滤下拉；高级设置折叠区 |
| D-2 | 打包 exe | 低优先级，写 build.bat + requirements.txt 备用 |
